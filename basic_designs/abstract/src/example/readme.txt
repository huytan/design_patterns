	In a typical organization, it is very common to have employees with different designations. This
can be represented in form of a class hierarchy with a base Employee class and a set of subclasses
each corresponding to employees with a specific designation.
 	Let us consider the following operations as part of designing the representation of an employee.

	1. Save employee data
	2. Display employee data
	3. Access employee attributes such as name and ID
	4. Calculate compensation
